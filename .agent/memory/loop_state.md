# Loop State — 當前任務進度（硬性上限機制）

> 這是跨 Session 的「我做到哪了」檔。代理會忘，倉庫不會忘。
> **硬性上限**：單檔 ≤50 行 / ≤2KB。超過必須蒸餾（見下方協議）。
> **任務結束**：清空本檔內容（保留本標題區），將蒸餾後的教訓歸入 `lessons.md`。

## 蒸餾協議（防止 loop_state 膨脹）
1. **每完成一個子任務**：用 ≤3 行記錄「做了什麼 / 改了哪些檔 / 下一步」。禁止記錄完整 diff。
2. **單檔達 40 行**：觸發中段蒸餾——把已完成子任務壓縮成 1 行摘要，只保留「未完成子任務」的詳細狀態。
3. **任務全部完成**：全檔蒸餾為 1 條 lesson（≤2 行）寫入 `lessons.md`，然後清空本檔。
4. **任務中斷（Session 結束但未完成）**：保留本檔，下次 Session 開頭讀本檔接續。
5. **禁止記錄**：完整程式碼、完整錯誤日誌、完整 diff——這些在 git 裡，不在這裡。
