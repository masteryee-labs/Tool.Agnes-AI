## Что такое Agnes AI?

**Agnes AI — это открытый десктоп AI-агент для программирования, написанный на чистом Rust с нативным GUI на egui (ноль Chromium / WebView2).** Он запускает **конвейер валидации с нулевым доверием из 22 агентов**, так что каждое действие, предложенное языковой моделью, перепроверяется детерминированными защитными шлюзами прежде, чем оно коснётся вашей файловой системы, оболочки или сети. Также в комплект входит **автономный цикл, управляемый целью** (Discover → Plan → Execute → Verify → Iterate), с **архитектурой суб-агентов** (Planner / Generator / Evaluator) и **изоляцией через Git worktree** для безопасного параллельного выполнения.

Agnes AI — это **бесплатная, локально-ориентированная альтернатива Claude Code, Cursor, Aider и Continue.dev** — ваш API-ключ и код никогда не покидают вашу машину, бинарник крошечный, запуск мгновенный, а UI — это минималистичное тёмное нативное приложение (без встроенного браузера, без Electron).

> **В одной строке:** Высокозащищённый, высокоскоростной нативный десктоп AI-агент на Rust, который никогда не доверяет устному заявлению модели «сработало» — только Exit Code 0 и пустой stderr считаются успехом.

## Ключевые возможности

### Базовые возможности
- **Нативный GUI на Rust** — eframe/egui + wgpu, без встроенного браузера, мгновенный запуск, крошечный объём
- **Минималистичный тёмный UI** — чистая чёрно-белая палитра в духе Claude Code / Codex / Devin / Antigravity 2.0; никаких отвлекающих фирменных цветов
- **Тихое выполнение** — все дочерние процессы (команды оболочки, компилятор, git, MCP-серверы) запускаются с `CREATE_NO_WINDOW` на Windows; на рабочем столе не всплывают окна CMD/PowerShell

### Рабочие пространства
- **Двойной режим Project / Global** — вкладки боковой панели переключаются между:
  - **Projects**: создание проекта из любой папки; каждая сессия чата вложена в свой проект; разговоры сохраняются в SQLite и возобновляются ровно с того места, где вы остановились
  - **Global**: отдельная вкладка для работы по всему компьютеру, где каждое действие требует подтверждения для каждого элемента

### Автономный цикл (Phase 5)
- **Цикл, управляемый целью** — задайте цель и условие выхода; он самостоятельно выполняет Discover → Plan → Execute → Verify → Iterate, пока условие не будет выполнено или не достигнут лимит итераций
- **Архитектура суб-агентов** — три независимые роли с отдельными промптами и состоянием диалога:
  - **Planner** — разбивает цель на атомарные подзадачи
  - **Generator** — реализует одну подзадачу за запуск, вызывая инструменты `write_file` / `run_command`
  - **Evaluator** — независимо проверяет вывод Generator; отвергает заявления об «успехе» только на словах
- **Изоляция через Git worktree** — каждый суб-агент Generator работает в изолированном git worktree + ветке; параллельные суб-агенты никогда не наступают на файлы друг друга; завершённая работа сливается обратно в основную ветку
- **Межсессионная память** — уроки, подводные камни и состояние цикла сохраняются в `.agent/memory/`, так что агент продолжает с того места, где остановился, между сессиями

### Безопасность и валидация
- **Конвейер валидации из 22 агентов** — каждый вызов инструмента от модели перепроверяется детерминированными шлюзами (ограничение путей, обнаружение shell-инъекций, сканирование утечек секретов, аудит AI-мусора, …) с вето одним голосом
- **Выравнивание песочницы** — записанные `.rs`-файлы немедленно компилируются (а их тесты выполняются); «заявляет об успехе, но не компилируется» отвергается на месте
- **Песочница WASM** — недоверенный код выполняется через интерпретатор `wasmi` на чистом Rust с пустым линкером (нет хост-импортов → нет I/O/сисвызовов/сети) и учётом топлива
- **Песочница Docker** — задачи уровня компиляции выполняются в контейнере с `--network=none`, `--rm`, рабочее пространство смонтировано в `/work`; векторизованные аргументы (без оболочки)
- **Нет доверия на словах** — Exit Code == 0 и пустой stderr — единственное определение успеха; устное заявление модели «сработало» никогда не принимается на веру

### Совместимость
- **Совместимость с Claude Skills** — поместите файлы `SKILL.md` в `.claude/skills/<name>/` в вашем рабочем пространстве; вызывайте их, вводя `/name` в чате. Правила проекта из `CLAUDE.md` загружаются автоматически
- **Совместимость с Claude MCP** — поместите стандартный `.mcp.json` в корень рабочего пространства или добавьте серверы в Settings → MCP Servers; списки инструментов подключённых серверов автоматически предоставляются модели

### Производительность
- **Многоуровневая память** — скользящее окно разбиения на фрагменты + 3-этапная воронка RAG над индексом FTS5 с водяными знаками дистилляции, чтобы избежать повторного сжигания токенов
- **Ограничение скорости и защита 20 RPM** — один глобальный общий токен-бакет ограничивает каждый API-вызов (включая дистилляцию и извлечение); `acquire()` ждёт пополнения, а не отклоняет, поэтому всплески никогда не пробивают лимит бесплатного тарифа в 20 запросов/минуту. При 429 клиент применяет экспоненциальную отсрочку на основе множителя. Каждый параметр управляется конфигом (`max_rpm`, настройки отсрочки повторов) — никаких магических чисел
- **Ротация нескольких API-ключей** — ротация между ключами нескольких аккаунтов (по счётчику + принудительное переключение при HTTP 420/429), чтобы оставаться полностью бесплатным, не упираясь в лимит одного аккаунта
- **Экономия токенов** — бюджет токенов на сессию с жёсткой блокировкой, живой индикатор бюджета в строке заголовка. Количество запросов сокращено по замыслу: Этап 0 делает локальный поиск в памяти FTS5, и при попадании полностью пропускает API-вызов извлечения (0 API-вызовов), а Этап 1+2 воронки RAG объединены в один вызов (2 вызова → 1)

## Часто задаваемые вопросы (FAQ)

### Agnes AI бесплатен?
Да. Agnes AI — открытое ПО (MIT) и бесплатен. Вы используете свой собственный API-ключ (например, ключ Agnes / OpenAI-совместимый). Функция ротации нескольких ключей позволяет комбинировать несколько аккаунтов на бесплатном тарифе, чтобы полностью избежать лимитов запросов.

### Agnes AI отправляет мой код в облако?
Сам Agnes AI работает на 100% локально. Ваш код никогда не ретранслируется через какой-либо сервер Agnes AI. Единственный сетевой трафик — это прямые API-вызовы к вашему LLM-провайдеру, которые вы настраиваете (это необходимо для любого агента на базе LLM). Ваш API-ключ хранится в `config.local.toml` (git-ignored) и никогда не попадает в систему контроля версий или в контекст модели.

### Чем Agnes AI отличается от Claude Code / Cursor / Aider?
- **vs Claude Code**: Agnes AI — открытое ПО, имеет нативный GUI (не только терминал), добавляет конвейер валидации с нулевым доверием из 22 агентов, изоляцию через Git worktree для параллельных суб-агентов и песочницы WASM/Docker.
- **vs Cursor**: Agnes AI — отдельное нативное приложение (без Electron/Chromium), открытое, с автономным циклом, управляемым целью, и архитектурой суб-агентов. Cursor — это форк VS Code.
- **vs Aider**: Agnes AI имеет полноценный GUI, автономный цикл, архитектуру суб-агентов, поддержку MCP/Skills, многоуровневую RAG-память и песочницы. Aider — только терминал без автономного цикла.
- **vs Continue.dev**: Agnes AI — отдельное приложение (не плагин IDE), с автономным циклом, суб-агентами и валидацией с нулевым доверием. Continue.dev — расширение для VS Code/JetBrains.

### Могу ли я использовать свой API-ключ?
Да. Вставьте ключ в Settings → API & Models или задайте его вручную в `config.local.toml`. Также можно указать несколько ключей (`keys = ["sk-a", "sk-b", "sk-c"]`) для ротации.

### Agnes AI поддерживает MCP (Model Context Protocol)?
Да. Agnes AI совместим с форматом Claude `.mcp.json`. Поместите стандартный `.mcp.json` в корень рабочего пространства или добавьте серверы в Settings → MCP Servers. Списки инструментов подключённых серверов автоматически предоставляются модели.

### Agnes AI поддерживает Claude Skills?
Да. Поместите файлы `SKILL.md` в `.claude/skills/<name>/` в вашем рабочем пространстве и вызывайте их, вводя `/name` в чате. Правила проекта из `CLAUDE.md` загружаются автоматически.

### Какие платформы поддерживает Agnes AI?
Agnes AI собирается на Windows, macOS и Linux (любая платформа, поддерживаемая Rust + egui). Мобильные привязки (iOS/Android) доступны через cargo-флаг `mobile` с помощью UniFFI.

### Agnes AI — открытое ПО?
Да, выпущен под лицензией MIT.

### На каком языке написан Agnes AI?
На чистом Rust, с использованием eframe/egui для нативного GUI, rusqlite для состояния, reqwest для HTTP и wasmi для песочницы WASM. Никакого JavaScript, никакого Electron, никакого Chromium, никакого WebView2.

### Есть ли в Agnes AI автономный режим?
Да. Переключитесь в **Goal mode** (💬 Chat → 🎯 Goal), опишите цель и условие выхода, нажмите Start. Цикл выполняется автономно: Planner разбивает цель, Generator реализует каждую подзадачу, Evaluator проверяет каждую из них. Остановить можно в любой момент.

## Настройка

Все локальные настройки хранятся в `config.local.toml` в корне репозитория (создаётся автоматически, **git-ignored** — ваш API-ключ никогда не попадает в систему контроля версий).

Самый простой путь — страница Settings в приложении (⚙ на боковой панели):

1. **Settings → API & Models** — вставьте ваш API-ключ, нажмите **Save**. Страница показывает замаскированную копию сохранённого ключа (`sk-xx…xxxx`), его отпечаток и зелёный «Saved ✓», так что вы всегда знаете, что активно.
2. **Settings → MCP Servers** — нажмите **+ Add Server**, заполните name / command / args; сервер запускается немедленно и сохраняется в конфиг.
3. **Settings → Skills** — показывает все навыки, обнаруженные в текущем рабочем пространстве.

Ручной эквивалент в `config.local.toml`:

```toml
[api]
key = "{{API_KEY}}"
# Optional: multiple keys for rotation (free-tier friendly)
keys = ["sk-a", "sk-b", "sk-c"]
key_rotation_every = 15
model = "agnes-2.0-flash"
session_budget = 500000

[[mcp_servers]]
name = "filesystem"
command = "npx"
args = ["-y", "@modelcontextprotocol/server-filesystem", "C:\\data"]
```

### MCP в формате Claude (`.mcp.json` в вашем рабочем пространстве)

```json
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "C:\\data"],
      "env": { "LOG_LEVEL": "info" }
    }
  }
}
```

### Skills в формате Claude

```
your-project/
└── .claude/
    └── skills/
        └── deploy/
            └── SKILL.md   # YAML frontmatter: name + description, then instructions
```

Введите `/deploy …` в чате для вызова. Skills и правила `CLAUDE.md` детерминированно внедряются в системный промпт (без дополнительных API-вызовов).

## Использование

### Режим чата
1. **Создайте проект** — боковая панель → вкладка Projects → **+ New Project**, выберите папку.
2. **Чат** — введите задачу; новая сессия создаётся под активным проектом и сохраняется. Нажмите на любую сессию на боковой панели, чтобы возобновить её позже с полной историей.
3. **Глобальный режим** — переключитесь на вкладку **Global** для работы вне папок проектов. Каждое действие отображается в правой панели для явного поэтапного подтверждения.
4. **Наблюдайте за агентами** — правая панель показывает все 22 валидирующих агента и их вердикты PASS/REJECT для каждого шага; ожидающие вызовы инструментов ждут там вашего Approve/Reject.

### Режим цели
1. **Переключитесь в Goal mode** — нажмите переключатель-капсулу вверху центральной панели (💬 Chat → 🎯 Goal).
2. **Опишите цель** — введите, что нужно сделать, и условие выхода (например, `file:Docs/report.md exists`).
3. **Нажмите Start** — цикл выполняется автономно: Planner разбивает цель, Generator реализует каждую подзадачу, Evaluator проверяет каждую из них. Панель статуса обновляется в реальном времени (текущая фаза, счётчик итераций, оставшийся бюджет).
4. **Остановите в любой момент** — кнопка остановки немедленно прерывает цикл.

## Модель безопасности

- API-ключи хранятся только в `config.local.toml` (git-ignored); любая строка `sk-` в исходном коде — автоматическое вето
- Команды выполняются как векторы аргументов — без конкатенации строк оболочки
- Ограничение путей: файловые операции вне выбранного рабочего пространства отклоняются (режим проекта)
- Коды выхода и stderr перехватываются в исходном виде; устное заявление модели об «успехе» никогда не принимается на веру
- Глобальный ограничитель скорости плюс экспоненциальная отсрочка при 429 защищают ключ и аккаунт от блокировки по лимитам запросов; ни одна подсистема (включая архивацию памяти) не может обойти общий лимит 20 RPM
- Сенсоры безопасности, выровненные по OWASP Top 10 (валидация ввода, SQL-инъекции, инъекции команд, обход путей, хардкод секретов, повышение привилегий, XSS, CSRF, небезопасная десериализация, утечка в логи)

## Дорожная карта

Agnes AI следует поэтапной дорожной карте. **Phase 0–5 завершены:**

- **Phase 0–1**: Базовый агент, нативный GUI, состояние в SQLite
- **Phase 2**: Конвейер валидации из 22 агентов
- **Phase 3**: Многоуровневая память + воронка RAG на FTS5
- **Phase 4**: Параллельная диспетчеризация, песочницы WASM/Docker, мобильные привязки, мультимодальность
- **Phase 5**: Движок автономного цикла, архитектура суб-агентов, изоляция через Git worktree, тихое выполнение

Полная дорожная карта — в [`Docs/08_ROADMAP.md`](Docs/08_ROADMAP.md).
